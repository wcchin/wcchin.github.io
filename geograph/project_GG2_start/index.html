Project GeoGraph-II: The starting point

Tuesday, October 20, 2015 » geography, graph, GUI, map, tool

I’ve been working on my-own new project, the project GeoGraph-II, which is in my mind for about more than one year.

The name

The name “GeoGraph-II” (read as ‘geography’), means that this project is about to build a tool, which integrate the geographical space and the network space. So the “GEO” and the “GRAPH”. And the two “I” behind it, the “~phy”, is also the greek number for two, sounds like “tool”. So, the “GEO”, the “GRAPH”, and the “TOOL”, that read as “geography~”.

The tool

As its name suggested, this tool planning to build for analyzing the geospatial network. Geospatial network is a form for representing the connections or relationships between spaces. Although the network analysis toolbox is already lying in the ArcGIS toolbox, the routing algorithm is ready in many database service or GIS-related software, the graph-theory and network structures are not included in current GIS. So, the major concept of building this tool is not for solving the routing/location-allocation/service-area types of problems. It is for analyzing the connected spaces.
